# 大模型API接口网关设计稿

用于对接不同的大模型 API，统一入口，支持不同协议直接的转化

## 功能需求

- 服务支持 openai 和 anthropic 协议的请求
- 上游模型暂时只支持 openai 和 anthropic 协议
- openai 协议时，需要支持 wire_type chat 与 responses 格式的双向转化
- 需要支持不同工具调用协议的转化，如 minimax <-> openai
- 请求上游模型支持失败重试
- 支持日志记录token消耗
- 设计上要支持可扩展，方便后续支持更多协议

## 配置

上游模型配置设计：

```toml
# 配置模型提供商
[model_providers.provider1]
base_url = "https://api.your-.com/v1"
env_key = "YOUR_PROVIDER_API_KEY"
modules = ['model1', 'model2']  # 提供商支持的所有模型
api_protocol = "openai"
wire_api = "chat"

[model_providers.provider2]
base_url = "https://api.your-.com/v1"
env_key = "YOUR_PROVIDER_API_KEY"
modules = ['model1', 'model2']
api_protocol = "anthropic"
```

客户端配置模型时，使用 提供商/模型名 的形式，如 provider1/model1

url 设计上，带上模型协议，以请求相应的协议，比如客户端需要 openai 协议，则请求 http://127.0.0.1/openai/
